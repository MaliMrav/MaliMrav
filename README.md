## Hi, I'm Vladimir Lekic 👋

I'm an architect, engineer, mentor, and lifelong student of systems-both technical and human.

Throughout my career I've had the privilege of working across networking, enterprise architecture, software engineering, infrastructure, and embedded systems. Those technologies have changed over the years, but one thing has remained constant:

**The most valuable thing we can build isn't software. It's understanding.**

I believe great engineering isn't measured by how clever a solution is, but by how naturally it fits the problem it solves. The best systems don't demand to be understood-they invite understanding.

That belief has shaped the way I approach every project.

## My Philosophy

I'm fascinated by architecture, not because architecture is complicated, but because good architecture makes complexity disappear.

Three principles guide almost everything I build:

- **Good architecture should feel inevitable.**
- **Architecture precedes implementation.**
- **Aliases are for muscle memory. Commands are for architecture.**

Those principles influence everything from embedded firmware and Linux tooling to infrastructure and software design.

I enjoy building systems that are coherent, discoverable, and maintainable-systems that teach as much as they solve.

## Why This GitHub Exists

The repositories you'll find here aren't intended to showcase clever code or the latest technology.

They're explorations.

Each project is an opportunity to share architectural thinking, document engineering decisions, and demonstrate that solving problems is often less about writing code and more about asking the right questions.

If someone reads one of my projects and becomes a better engineer because of it, then the project has achieved its purpose.

## Current Projects

### Telemetry

A teaching platform disguised as an embedded systems project.

Telemetry explores architecture, software design, UX, and engineering principles using ESP32-based hardware as the medium-not the destination.

### DynDNS Update

A clean, architecture-first implementation of a Dynamic DNS updater.

Originally created to solve a practical problem, it has grown into an exploration of maintainable Python architecture, configuration management, and operational simplicity.

### Bridge _(in development)_

An opinionated command architecture for Linux.

Bridge isn't about replacing the command line. It's about making it more discoverable.

Its goal is to help engineers express intent rather than remember syntax.

## Engineering Principles

Across every project you'll see the same recurring ideas:

- Build systems that teach.
- Hide complexity. Never capability.
- Discoverability is a feature.
- Consistency beats cleverness.
- Coherence over completeness.
- Every design decision should answer one question:

**Does this feel inevitable?**
